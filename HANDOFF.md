# 운동 코치 앱 — 개발 핸드오프 문서

> 다른 개발자(AI 포함)가 이 문서만 읽고 바로 이어서 개발할 수 있도록 정리한 문서.
> 기준 시점: v0.10 (versionCode 10). 작성일: 2026-06-14.
> ⚠️ 주의: 사용자가 준 질문 템플릿은 원래 "학습/오답 진단 앱"용이라 일부 항목(학년·과목·오답진단·유사문제)은 **이 앱과 무관** → "해당 없음"으로 표기.

---

## 1. 앱 한 줄 요약

**삼성 헬스(Galaxy Watch/폰) 운동 기록을 Health Connect로 자동 수집해, Gemini/Claude AI가 친한 친구 같은 말투로 분석·코칭하고 다음 훈련을 처방해주는 안드로이드 종합 운동 코칭 앱.** 달리기·헬스·걷기·자전거·등산 지원. 실시간 음성 러닝 코치 포함. (제작사: 두나무)

## 2. 핵심 기능 목록

### ✅ 합의 + 구현 완료 (v0.10 기준)
- **운동 자동 수집**: Health Connect에서 운동 세션 읽기 (삼성헬스가 HC에 기록한 데이터)
- **운동 종류**: 달리기 / 헬스·근력 / 걷기 / 자전거 / 등산
- **캘린더**: 월 단위, 운동한 날에 종류별 색 점, 날짜 선택 시 그날 운동만
- **운동별 필터**: 전체/종류별 모아보기 (삼성헬스 스타일 상단 칩)
- **운동 리스트 + 상세**: 거리·시간·평균페이스·평균/최고 심박·칼로리·고도, **시작~종료 시각**, **0.5km 구간 페이스**(상세 열 때 지연 로딩), 데이터 **출처(앱·기기)**
- **AI 코칭(운동별)**: 친구 말투, 구조화 출력. 직전 처방과 비교("지난 처방대로 했어?") 컨텍스트 포함
- **카메라/갤러리 사진 → AI 비전 분석** (기구·자세·식단 등)
- **AI 엔진 선택**: Gemini(무료)/Claude(유료), 앱 내 키 입력·저장, "키 연결 테스트" 버튼
- **실패 피드백 토스트** 전반 적용
- **라이브 러닝 코치**: 실제 GPS 또는 시뮬레이션, **TTS 음성 코칭**(목표 페이스 대비 빠름/느림/유지), km마다 AI 한마디(옵션), 종료 후 요약+코칭. 화면 켠 상태 기준(백그라운드 X)
- **레벨**: Lv1 5km 완주 / Lv2 5km 36분 / Lv3 10km 50분 (달리기)
- **중복 처리**:
  - 합산 컨테이너(다른 기록 2개+ 시간상 포함) **자동 숨김**
  - **중복 의심 질문**(같은 종류+시작 5분 이내): 상세정보(시간·페이스·심박·칼로리·출처) 보여주고 ⌚워치 기록 추천 힌트와 함께 "어느 걸 숨길지" 선택
  - 운동 상세에서 **수동 숨기기**
  - 숨김은 **앱 로컬(SharedPreferences)** — 삼성헬스 원본은 안 건드림(되돌리기 가능)
- **코칭 결과 자동 스크롤**

### ⏸ 보류 / 미구현 (논의됨, 아직 안 만듦)
- **하루 종합 코칭** (개별 운동 합산 분석) — 추가하기로 제안, 미구현
- **통계 그래프** (주/월 추이) — 미구현
- **운동 직접 추가** — 미구현
- **코칭 히스토리 모아보기** — 미구현
- **새 운동 자동 감지 알림** (WorkManager): 초기 프로토타입(RunCheckWorker) 있었으나 피벗 중 제거됨. 재구현 필요. **백그라운드 GPS(포그라운드 서비스)** 는 데모라 일단 제외
- **라이브 코치 목표 다양화** (거리/시간/칼로리/인터벌/심박존 등 갤럭시워치처럼) — 요청됨, 미구현
- **로고 + 스플래시(1일 1회) + 앱 이름 확정** — 미정/미구현
- **개인정보 처리방침 + 첫 실행 동의 화면** — 신체정보 수집하므로 필요, 미구현
- **앱 내 자동 업데이트** — 논의됨, 미구현

### ❌ 불가능으로 결론난 것
- **삼성헬스/Gemini/Claude 구독 로그인(OAuth)으로 API 키 자동 발급** — 불가 (구독≠API). Gemini는 구글 OAuth로 기술상 가능하나 Cloud 프로젝트+동의화면 필요해 더 복잡 → 비추천
- **갤럭시 워치 "목표" 자동 가져오기** — 불가 (Health Connect는 완료된 운동 데이터만 노출, 사용자가 설정한 목표는 노출 안 함)
- **앱에서 삼성헬스 데이터 진짜 삭제** — 사실상 불가 (HC는 삼성헬스의 거울, 지워도 재동기화됨) → "숨기기"로 대체
- **웹에서 실제 삼성헬스 데이터 읽기** — 불가 (Health Connect는 안드로이드 전용). 웹은 프로토타입(목업 데이터)용

## 4. 핵심 사용자 시나리오
1. **운동 후 코칭**: (워치로 운동) → 삼성헬스가 Health Connect에 기록 → 앱 열기 → 캘린더/리스트에서 해당 운동 탭 → 상세(거리·페이스·심박·구간) 확인 → (선택) 메모/사진 추가 → "코칭 받기" → AI가 평가+다음 훈련 처방
2. **사진 첨부 분석**: 운동 상세에서 📷촬영/갤러리 → AI가 사진(기구·자세·식단)까지 함께 분석
3. **실시간 러닝**: 🔴라이브 → 목표 페이스 입력 → GPS/시뮬 시작 → 달리는 중 음성으로 "속도 올려/유지/빠름" + km마다 알림 → 종료 → 요약 + 전체 코칭
4. **중복 정리**: 앱 로드 시 중복 의심 기록 발견하면 팝업 → 워치 기록 추천 힌트 보고 폰 기록 숨기기
5. (보류) **다음날 비교**: 새 기록 들어오면 지난 처방 대비 평가

## 5. 기술 스택 / 플랫폼 결정 사항
- **플랫폼**: 안드로이드 네이티브 (실시간 자동 연동이 1순위라 웹 대신 앱 선택). 웹은 프로토타입 용도만.
- **언어/UI**: Kotlin + Jetpack Compose (Material 3), 다크 테마, 그린 액센트 `#3DA58A`
- **SDK**: minSdk 28, targetSdk 34, compileSdk 34
- **빌드**: AGP 8.5.2, Gradle 8.9, Kotlin 1.9.24, Compose BOM 2024.06.00, Compose compiler 1.5.14, JDK 17 (Temurin)
- **건강 데이터**: `androidx.health.connect:connect-client:1.1.0-alpha07` (ExerciseSession/HeartRate/Distance/Speed/Steps/Calories/Elevation)
- **AI**:
  - Gemini(무료): `gemini-2.5-flash` → 실패 시 `gemini-2.5-flash-lite` 폴백. ⚠️ `gemini-2.0-flash`는 무료 할당량 0이라 쓰면 안 됨. 엔드포인트 `generativelanguage.googleapis.com/v1beta/models/{model}:generateContent?key=`

- **기타 의존성**: WorkManager 2.9.0(현재 미사용/워커 제거됨), kotlinx-coroutines
- **백엔드/DB**: 없음. 전부 로컬. `CoachStore`(SharedPreferences)에 API키·provider·숨김목록·해결된중복·운동별 코칭/메모 저장

## 6. AI 프롬프트 / 출력 형식 합의 사항
- 템플릿의 "오답 진단/유사문제 생성" = **해당 없음** (운동 코칭 앱)
- **말투**: 친한 운동 코치 친구, **반말**, 격려하는 톤, 이모지 적당히
- **출력 형식**(제목 그대로 유지):
  - `### 1. 오늘 평가`
  - `### 2. 다음 훈련 처방`
    - 달리기: 인터벌/템포런/롱런 중 선택 + 거리·세트·휴식 구체적
    - 헬스: 다음 분할/종목·중량·세트·휴식
    - 기타: 다음 목표 거리·시간·강도
  - 달리기: `### 3. 구간별 목표 페이스`, `### 4. 심박수 목표`
  - 헬스: `### 3. 자세·폼 팁`, `### 4. 강도 가이드`
  - 기타: `### 3. 페이스·심박 가이드`
  - 공통: `### 5. 준비운동`, `### 6. 보조 운동`
  - 끝에 "핵심만, 너무 길지 않게."
- **비교 코칭**: 직전 처방 + 직전 기록을 함께 넣어 "지난 처방대로 했는지" 평가 (새 기록일 때)
- **라이브 km 응원**: "딱 한 문장, 20자 내외, 힘차게"
- **목표 컨텍스트**: 달리기는 "10km 50분 이내(5:00/km), 현재 약 5:22/km" + 레벨 상태 주입
- **사진 있을 때**: "첨부 사진도 함께 분석(기구/자세/장소/식단)"

## 7. UI / 화면 구성 합의 사항
- **디자인**: 다크 테마, 그린 액센트, 삼성헬스 UI 일부 차용(상단 운동 필터 + 캘린더)
- **화면/구성**:
  - 메인: 상단(타이틀 + 🔴라이브 + ⚙️설정) → 운동별 필터 칩 → 월 캘린더(종류별 점) → 운동 리스트
  - 운동 상세: 바텀시트(통계 그리드, 구간 페이스, 메모, 사진, 코칭 버튼/결과, 숨기기)
  - AI 설정: 다이얼로그(Gemini/Claude 토글, 키 입력, 키 테스트)
  - 중복 의심: 다이얼로그(두 기록 상세 + ⌚워치 추천 + 숨기기/둘다유지)
  - 라이브: 별도 화면(설정→달리는중 큰 숫자→요약)
- **이름 이슈**: 코드상 표시 타이틀 "운동 코치", strings.xml `app_name`은 "러닝 코치"로 불일치 → 통일 필요. 정식 이름 **미정**(아이디어: 런데이류, 두나무=나무 테마, 종합운동 고려)

## 9. 제약사항
- **언어**: 한국어 전용 (UI/프롬프트 모두 한국어)

- **일정/팀 규모**: **미논의**

## 10. MVP 범위 (1차)
**현재 v0.10이 사실상 MVP** — 아래까지 구현됨:

## 부록: 알아두면 좋은 함정(실제 겪은 이슈)
- 설치 안 됨 → 99% **Auto Blocker** 때문. 끄면 됨. (충전전용 USB 케이블, USB디버깅 못 찾음 등도 겪음 → 결국 adb로 해결)
- Health Connect 권한: 매니페스트 선언 권한과 코드 요청 권한이 **일치**해야 함(불일치 시 "모두 허용해도 부족" 버그). 현재 핵심 5개(운동·심박·거리·속도·걸음) 필수 + 칼로리·고도 선택
- 삼성헬스→Health Connect **공유(쓰기) 설정**을 켜야 데이터가 들어옴
- 성능: 운동마다 구간(속도·심박 샘플) 계산하면 로딩이 매우 느림 → **구간은 상세 열 때만 지연 계산**(v0.10에서 수정)
- 멀티스포츠(브릭) 시 삼성헬스가 **합산 컨테이너 기록**을 추가로 만듦 → 시간상 2개+ 포함하면 자동 숨김
- 워치 vs 폰 중복: **워치 기록(심박 있는 것) 유지** 권장
